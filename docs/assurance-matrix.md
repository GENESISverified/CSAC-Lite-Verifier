# Assurance Matrix

The assurance matrix helps organizations see how the review layers work together.

## Matrix

| Question | Layer | Review output |
|---|---|---|
| What is being reviewed? | Review Package | Submitted outcome |
| What is the current state? | Status Review | Review status |
| Why does it still matter? | Decision Context | Context note |
| Who can question it? | Human Review Layer | Reviewer note |
| What remains after review? | Governance Trail | Decision record |

## Purpose

The matrix keeps the review process readable.

It helps teams avoid mixing technical status, human judgement and governance record into one unclear object.

## Principle

A strong review process separates the questions before it combines the decision record.
